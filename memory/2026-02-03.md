## Session 3: Cloudinary Profile Photo Migration ☁️

### ✅ Notifications Review

### ✅ Profile Photo → Cloudinary Migration

**Problem:** Photos stored as Base64 data URLs directly in MongoDB
- 16MB document limit risk
- 33% overhead from Base64 encoding
- No CDN caching, slow to load

**Solution:** Cloudinary integration (free tier: 25GB storage, 25GB bandwidth/month)

**Backend Changes:**
| File | Change |
|------|--------|
| `package.json` | Added `cloudinary` dependency |
| `src/services/cloudinaryService.js` | **NEW** — Upload, delete, URL helpers |
| `src/controllers/userController.js` | `updateProfile` + `updateProfilePicture` now upload to Cloudinary |
| `src/controllers/authController.js` | Registration uploads photo to Cloudinary |
| `.env.example` | Added `CLOUDINARY_*` config variables |
| `scripts/migrate-photos-to-cloudinary.js` | **NEW** — Batch migration script for existing photos |

**Features:**
- Auto image optimization (max 800x800, quality auto, WebP for supported browsers)
- Old photos deleted from Cloudinary when replaced
- Graceful fallback — registration continues even if upload fails
- Migration script processes in batches with rate limit protection

**Flutter Changes:**
- None needed! `profile_card.dart` already handles HTTP URLs via `CachedNetworkImage`

### 🔧 Deployment Steps
1. Create free Cloudinary account: https://cloudinary.com/users/register_free
2. Add to `.env.local`:
   ```
   CLOUDINARY_CLOUD_NAME=your_cloud_name
   CLOUDINARY_API_KEY=your_api_key
   CLOUDINARY_API_SECRET=your_api_secret
   ```
3. Run migration for existing photos:
   ```bash
   cd manas-backend-new-main/manas-backend-new-main
   node scripts/migrate-photos-to-cloudinary.js
   ```
